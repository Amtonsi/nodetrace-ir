NODETRACE IR 0.3.0 — ЗАГРУЗОЧНЫЙ x86 WINPE НОСИТЕЛЬ
=======================================================

Назначение
----------
Этот вариант является загрузочным WinPE ISO. Он исследует отключённую
установку Windows и хранит кейс на отдельном writable-томе.

Перед загрузкой подключите:
  1. исследуемый диск с Windows;
  2. отдельный USB/диск для доказательств;
  3. по возможности создайте в корне evidence-диска пустой файл
     NODETRACE_EVIDENCE_VOLUME.

Автозапуск
----------
После загрузки startnet.cmd вызывает wpeinit и launch_nodetrace.cmd.
Launcher сам находит Windows по реестровым hive-файлам, выбирает отдельный
writable evidence-том и запускает NodeTraceIR.exe с параметрами --winpe,
--offline-root и --data-dir. Кнопки запуска анализа нет и не требуется.

Если Windows-том защищён BitLocker, сначала разблокируйте его и снова запустите:

  %SystemRoot%\System32\launch_nodetrace.cmd

Результат
---------
Данные создаются в:

  <evidence>\NodeTraceIR-Evidence\session-*\

После анализа каталог exports содержит проверяемый ZIP. Рядом сохраняются
report.html, масштабируемый graph.svg, impact.json и SHA256SUMS.txt.

Границы
-------
Офлайн-режим не видит процессы, память, TCP/DNS volatile-состояние отключённой
Windows. Точный URL или USB ID выводится только при наличии прямого
сохранившегося артефакта. Историческая запись об USB без связи с файлом не
выдаётся за доказанный источник доставки.

AVZ
---
AVZ является x86-компонентом с отдельной лицензией и запускается только в
аналитическом профиле без лечения, удаления и карантина. База AVZ внутри ISO
закреплена хэшем и во время расследования через сеть не обновляется.

Обновление выполняется на доверенной build-машине командой
tools\update_avz_base.py с явным подтверждением лицензии и repin, после чего
ISO необходимо полностью пересобрать и снова проверить.

Никогда не храните реальные кейсы, вредоносные образцы или AVZ-бинарники в
публичном исходном репозитории NodeTrace IR.
